# DFS / BFS

> Graph 탐색, 하나의 노드를 시작으로 다수의 노드를 방문하는 것.

1. 스택: DFS, 기본리스트에서 append()와 pop() 사용

2. 큐: BFS, collections모듈의 deque사용. append()와 popleft() 사용

3. 재귀함수

   	- 스택자료구조 사용.

    - 반복문보다 더 간결함. (점화식을 그대로 소스코드로 옮겼기 때문)
      	- 점화식: 특정한 함수를 자신보다 더 작은 변수에 대한 함수와의 관계로 표현한 것을 의미

4. 인접행렬, 인접리스트: python에서는 2차원 리스트로 구현



### 1. DFS (Depth First Search, 깊이우선탐색)

> 특정한 경로로 탐색하다가 특정한 상황에서 최대한 깊숙이 들어가서 노드를 방문한 후, 다시 돌아가 다른 경로로 탐색하는 알고리즘 (노드 방문 = 스택에 들어간 순서)



### 2. BFS (Breath First Search, 너비우선탐색)

> 가까운 노드부터 탐색하는 알고리즘.
>
> 인접한 노드를 반복적으로 큐에 넣도록 알고리즘을 작성하면 자연스럽게 먼저 들어온 것이 먼저 나가게 되어, 가까운 노드부터 탐색을 진행하게 된다.



### 3. 실전문제

1. 음료수 얼려먹기
   * DFS (그래프형태로 모델링이 가능)
   * 부분그래프 개수찾기
   * DFS가 1회 종료되서 나올때마다 카운트해준다. 왜냐하면 DFS가 한 번 종료됐다는 것은 이어진 인접한 노드들로 이뤄진 부분그래프를 하나 찾아낸 것이기 때문이다.

2. 미로탈출
   	* BFS, 다익스트라, 벨만포드, 플루이드 사용가능
   	* DFS와 달리 BFS는 언제나 최적해를 보장한다.
